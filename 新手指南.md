## 1. 准备工作（只需做一次）

### 1.1 你电脑上需要有什么

| 软件                 | 怎么装                                                                                | 怎么验证装好了                                       |
| ------------------ | ---------------------------------------------------------------------------------- | --------------------------------------------- |
| **Python** (≥3.10) | [python.org](https://www.python.org/downloads/) 下载安装，**安装时勾选「Add Python to PATH」** | 终端输入 `python --version`，看到类似 `Python 3.12.x`  |
| **Git**            | [git-scm.com](https://git-scm.com/downloads) 下载安装，一路默认就行                           | 终端输入 `git --version`，看到类似 `git version 2.x.x` |
| **IDE（编辑器）**       | 推荐 [VS Code](https://code.visualstudio.com/)（免费），装好 Git 和 Python 插件                | 能打开文件夹、看到代码高亮就行                               |

### 1.2 克隆项目到本地

打开终端（Windows 用 PowerShell，Mac 用终端），找个你放项目的地方，把仓库克隆下来：

```powershell
# 先到你放项目的地方（比如桌面）
cd ~/Desktop

# 克隆仓库
git clone https://github.com/lc-Cao/XPHO_PROJECT

# 进入项目目录
cd mvp
```

<br />

### 1.3 用 IDE 打开项目

克隆完成后，用 VS Code（或你喜欢的编辑器）打开这个文件夹：

- **VS Code**：菜单 → 文件 → 打开文件夹 → 选中刚克隆的文件夹
- 或者终端直接敲：`code .`

打开后你会看到类似这样的目录结构：

```
├── .env.example
├── .gitignore
├── README.md
├── 新手指南.md                  ← 你现在看的这份文档
├── schema/
├── data/
│   └── Q1/                      ← 示例题目，可以参考
├── scripts/
│   ├── pipeline_new.py          ← 标注入口（你的主要操作对象）
│   └── ...（其他脚本）
```

### 1.4 安装依赖包

在 IDE 里打开终端（VS Code：菜单 → 终端 /新建终端），敲入：

```powershell
pip install openai python-dotenv matplotlib
```

看到 `Successfully installed ...` 就说明搞定了。

### 1.5 配置 LLM（可选，但强烈推荐）

如果你有 LLM 的 API Key（DeepSeek / OpenAI / ...），配置后能自动填考点、难度等字段，省很多时间(结果需要审核）。

没有也没关系——不配也能跑，只是要多手动填几个字段。

```powershell
# 复制配置模板
copy .env.example .env
```

然后在 IDE 里打开 `.env` 文件，改成：

```ini
# 把你自己的 key 填在这里
LLM_API_KEY=sk-你的密钥

# 如果用 DeepSeek（推荐，便宜好用），改下面两行
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

保存关闭。

> **没有 API Key？** 跳过这步。运行时加 `--no-llm` 就行。

## 2. 写题目（最核心的一步）

### 2.1 文件放哪里

每一道题放在 `data/` 下一个以题目编号命名的文件夹里。

```
data/
└── Q1/                  ← 你的题目文件夹（Q1、Q2、Q3...）
    ├── Q1_problem.md     ← 【核心】你要写的题目文件
    └── image/            ← 题目用到的图片，手动放进来
        ├── ipho_2025_1_1.png
        └── ipho_2025_1_2.png
```

> **编号规则**：Q1、Q2、Q3... 按顺序来。如果已有 Q1，你就创建 `data/Q2/`。

### 2.2 题目 MD 怎么写

用 Markdown 格式写，必须包含以下几个板块（按 `# 板块名` 一级标题来写），我们只需要修改不符合物理规律的内容即可。

模板

````markdown
# 题目名称

# Question
这里是题目的背景介绍。介绍物理场景、已知条件等。

### Part A: 第一板块
这是 Part A 的总述。

**A.1.** 这是子问题 A.1 的具体问题。

**A.2.** 这是子问题 A.2 的具体问题。

### Part B: 第二板块
这是 Part B 的总述。

**B.1.** 这是子问题 B.1 的具体问题。

**B.2.** 这是子问题 B.2 的具体问题。

# Answer
本题的标准解答。

### Part A: 第一板块

**[A.1's Standard Solution]**
这里是 A.1 的详细解答过程。可以用 LaTeX 公式，比如 $F = ma$。
$$E = mc^2$$

**[Final Result]** : $F = 10\text{ N}$

**[A.2's Standard Solution]**
这里是 A.2 的详细解答过程。

**[Final Result]** : $v = \sqrt{2gh}$

### Part B: 第二板块

**[B.1's Standard Solution]**
B.1 的解答过程。

**[Final Result]** : $P = 100\text{ W}$

**[B.2's Standard Solution]**
B.2 的解答过程。

**[Final Result]** : $T = 300\text{ K}$

# GradingRubric
评分标准，用表格写出。

| sub-part | item | marks | notes |
|----------|------|-------|-------|
| **A.1**  | correctly applies Newton's second law | 0.5 | $F = ma$ |
|          | obtains correct numerical result      | 0.5 | tolerance 5% |
| **A.2**  | correctly sets up energy conservation | 0.5 |             |
|          | derives correct velocity expression   | 0.5 | $v = \sqrt{2gh}$ |
| **B.1**  | correctly computes power               | 1.0 |             |
| **B.2**  | correctly computes temperature         | 1.0 |             |
| Part A Total |      | 2.0  |             |
| Part B Total |      | 2.0  |             |

# DiagramCode
如果需要自动生成图表，这里放 Python 代码。

```python
import matplotlib.pyplot as plt

fig, ax = plt.subplots()
ax.plot([0, 1, 2], [0, 1, 4])
ax.set_xlabel("x")
ax.set_ylabel("y")
ax.set_title("Example Figure")
plt.savefig("Figure_1.png")
````

# QuestionReview

（可选）题目审查意见。

# AnswerValidation

答案校验意见。

````

### 2.3 格式规则速查表

| 你要写的东西 | 必须这样写 | 例子 |
|-------------|-----------|------|
| **题目板块** | `# Question` | 一级标题，后面跟内容 |
| **解答板块** | `# Answer` | 同上 |
| **评分标准** | `# GradingRubric` | 后面用表格 |
| **图表代码** | `# DiagramCode` | 后面跟 ` ```python ` 代码块 |
| **子板块** | `### Part X: 名称` | 三级标题，如 `### Part A: Mechanics` |
| **子问题** | `**A.1.**` | 加粗 + 字母.数字 + 加粗 + 空格 |
| **解答标记** | `**[A.1's Standard Solution]**` | 加粗 + 方括号 + 标准格式 |
| **答案标记** | `**[Final Result]** :` | 加粗 + 冒号 + 答案 |

> ⚠️ **这些格式一个都不能错**，错了脚本解析不出来！

### 2.4 图片怎么处理

1. 把题目配图放进 `data/Q1/image/` 目录
2. 在 MD 中引用：`![](image/你的图片名.png)`
3. 图片命名建议：`源_年份_题号_序号.png`，如 `ipho_2025_1_1.png`

## 3. 运行标注

# 没有 API Key 的话，加上 --no-llm

python scripts/pipeline_new.py --file Q1 --source ipho --year 2025

## 4. 人工补全（查漏补缺）

### 4.1 打开待办清单

跑完脚本后，打开 `data/Q1/work/Q1.review.todo.md`，会看到类似这样的清单：

```markdown
# Review TODO

## Manual items (after pipeline)
- [ ] Fill 改造后问题 for A.1
- [ ] Fill 改造后答案 for A.1
- [ ] Fill 改造后答案类型 for A.1
- [ ] Fill 改造后答案单位 for A.1
- [ ] Fill 多解法标注 for A.1 (if applicable)
- [ ] Fill 错误解题步骤 for A.1 (if applicable)
...
- [ ] Manual physics correctness review
- [ ] Verify all values are in English
- [ ] Verify 关联图片路径 and 模态类型 per image
```

### 4.2 哪些必须填、哪些可以不填

| 字段          | 是否填写 | 怎么填                                                                                  |
| ----------- | :--: | ------------------------------------------------------------------------------------ |
| **改造后问题**   |   填  | 把原始问题改写为适合机器判题的格式，比如把「求力的大小」改为「$F =$ \_\_\_ N」                                       |
| **改造后答案**   |   填  | 对应改造后问题的答案，可以是表达式或数值                                                                 |
| **改造后答案类型** |   填  | 选一个：`expression` / `numerical` / `choice` / `equation` / `open-ended` / `inequality` |
| **改造后答案单位** |   填  | 如 `N`、`m/s`、`J`，没有就留空 `""`                                                           |
| 核心思路        |   填  | 如果没配 LLM，这栏是空的，用一句话写解题思路（英文）                                                         |
| 难度          |   填  | `easy` / `medium` / `hard`，没配 LLM 需要手动填                                              |
| 关联考点        |   填  | 三级分类，如 `["Mechanics", "Dynamics", "Newton's Second Law"]`                            |
| 多解法标注       |  可选  | 有其他解法就填，没有留空数组 `[]`                                                                  |
| 错误解题步骤      |  不填  | 现阶段留空 `[]`                                                                           |

<br />

### 4.3 实际操作

打开 `data/Q1/Q1_problem.json`，找到对应子问题的对应字段，直接编辑。

例如，给 A.1 补全改造后问题：

```json
{
  "标注基础信息": { ... },
  "题目信息": {
    ...
    "问题原文": "Find the tension force in the tether.",
    "改造后问题": "T = ___ N",       ← 填这里
    "最终答案": ["$T = 10\\text{ N}$"],
    "改造后答案": ["10"],            ← 填这里
    "答案类型": ["expression"],
    "改造后答案类型": ["numerical"],  ← 填这里
    "答案单位": ["N"],
    "改造后答案单位": ["N"],          ← 填这里
    ...
  }
}
```

> **注意**：JSON 格式很严格——逗号、引号、方括号一个都不能错。改完建议跑一下校验。

## 5. 验收与提交

### 5.1 校验 JSON 是否正确

```powershell
python scripts/validate_annotation_new.py data/Q1/Q1_problem.json data/Q1/Q1_problem.md data/Q1
```

期望输出：

```
[validate] 0 error(s), X warning(s)
```

- **0 error** = 格式合格 ✅
- warning 可以接受，主要是提醒你某些字段还没填
- 如果有 error，根据提示去修 JSON

### 5.2 把结果提交到 Git（保留痕迹）

```powershell
# 如果你之前运行 --git 了，这步已经自动做了
# 否则手动提交
python scripts/final_commit.py --file Q1
```

提交后可以查看历史：

```powershell
git log --oneline
```

会看到两条记录：

```
def5678 annotate: Q1 annotated per spec v0416    ← 标注版本
abc1234 upload: original Q1 before annotation    ← 原始版本
```

## 6. 常见问题

### Q1：运行时报错 `No module named 'openai'`

```powershell
pip install openai
```

### Q2：运行时报错 `input not found: D:\...\Q1_problem.md`

说明文件路径不对。检查：

- `data/Q1/Q1_problem.md` 是否存在
- 文件名是否严格是 `Q1_problem.md`（注意大小写、下划线）

### Q3：GradingRubric 解析不出来

检查表格格式：

- 表头行必须有 `| sub-part | item | marks | notes |`
- 分隔行必须有 `|----------|------|-------|-------|`
- 子问题标记必须是 `**A.1**` 这种格式
- 分数不要太大（>2.0 的会被当作小计行跳过）

### Q4：子问题没被识别

检查：

- 子问题标记必须是 `**字母.数字.**` 格式，如 `**A.1.**`
- 不能是 `**A1**` 或 `**A-1**` 或 `A.1.`
- `**` 和 `.` 一个都不能少

### Q5：答案没被提取出来

检查：

- 解答标记必须是 `**[A.1's Standard Solution]**`（注意单引号 `'s`）
- 答案标记必须是 `**[Final Result]** :`（冒号后面有空格也可以）
- LaTeX 公式用 `$...$` 或 `$$...$$` 包裹

### Q6：中文 Windows 下出现乱码

运行前设置编码环境变量：

```powershell
$env:PYTHONIOENCODING="utf-8"
$env:PYTHONUTF8="1"
```

或者每次运行前加上：

```powershell
$env:PYTHONIOENCODING="utf-8"; $env:PYTHONUTF8="1"; python scripts/pipeline_new.py --file Q1
```

### Q7：LLM 调用失败怎么办

几种可能：

- API Key 过期或余额不足 → 换 key 或充值
- 网络问题 → 检查是否要挂代理；如果在中国大陆用 DeepSeek，网络通常没问题
- 模型不支持 JSON mode → 换模型，推荐 `deepseek-chat` 或 `gpt-4o`
- **不想折腾 LLM** → 直接 `--no-llm` 跑，后续手动填

### Q8：如何标注 Q2、Q3……

参照 Q1 的结构创建即可：

```
data/
├── Q1/
│   ├── Q1_problem.md
│   ├── Q1_problem.json
│   └── image/
├── Q2/                          ← 新建
│   ├── Q2_problem.md            ← 放新题目
│   └── image/                   ← 放图片
└── Q3/
    ├── Q3_problem.md
    └── image/
```

然后运行：

```powershell
python scripts/pipeline_new.py --file Q2 --git
```
